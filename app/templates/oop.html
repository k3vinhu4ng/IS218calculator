Object Oriented Programming (OOP)

Encapsulation, Abstraction, Inheritance, Polymorphism

Introduction

Object Oriented Programming (OOP) is a pattern of programming that focuses on classes and objects. Programming done with OOP allows each object to have its own procedures and method. In addition, OOP focuses on the logic and relationships between these objects. Four main concerns of OOP is encapsulation, abstraction, inheritance, and polymorphism.

This article will give a brief introduction to OOP, and how OOP is used in a calculator program.

Encapsulation

Encapsulation describes wrapping data and methods that work on data within one unit. This restricts direct access to certain variables and methods, unless allowed to do so. To change the data within a certain object, it can only be done using the object's method. These variables within the object are known as private variables.

Encapsulation is very useful because it protects data from accidentally being changed.

Encapsulation in Calculator Program

We use encapsulation in our calculator program by creating classes that has it owns methods, variables, and functions. For example, we have classes for addition, subtraction, multiplication, and division. We also have a class for the history of calculations. The history of calculations is stored in a list, and the history calculation class has a function to clear the history. Encapsulation ensures that a function class, such as the addition class, cannot clear the history of the history class. The clear history function only works for the appropriate class that is was designed for.

Abstraction

Abstraction refers to the abstract classes that can be used as a blueprint for other classes. Abstract classes allow us to create a set of methods that can be used for any child classes built from the abstract class.

For example, we can create a class with an abstract method. Within this class, we can create other classes that have unique properties. However, we can use the abstract method of the parent class for each child class. Each child class has the abstract method despite being different classes.

Abstraction in Calculator Program

In the calculator program, we used abstraction for creating a base class that takes care of the input values for the calculator. This base class reduces redundancies and help provide many useful functions. The base class allows us to create an object, convert input values into a tuple, and convert the input values to floats. This base class is also used for our addition, subtraction, multiplication, and division classes. Rather than writing all these functions for every calculation class, we included it in the base class which the calculation class can use.

Inheritance

Inheritance is the ability of one class to inherit properties from another class. For example, child classes can inherit properties from parent classes.

Inheritance in Calculator Program

Inheritance is used a lot in our calculator program. Inheritance allows us to reuse fields and methods of existing classes. The addition, subtraction, multiplication, and division classes inherit the properties of the calculation class that allow for multiple inputs, changing list to tuples, changing numbers to floats, and creating objects.

Then, the calculation history class inherits properties from each calculation class, such as addition. For example, the history class now inherits the properties of the addition class because the history list includes the addition object, which has the result value and class type.

Lastly, the calculator class inherits all the properties of the classes mentioned before. This hides the complexity of the code behind one simple class.

Polymorphism

Polymorphism refers to having many forms. In programming, this means that the same function can be used for different types. For example, the Python function len() can be used for numbers, strings, and even lists.

Polymorphism in Calculator Program

A good example of polymorphism in the calculator program is the get_result() function. This function exists in the addition, subtraction, multiplication, and division class. Despite being different classes, the get_result() function can be used for all of them. This is very similar to how the Python function len() can be used for many forms, such as numbers, strings and lists.
